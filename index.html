Tema 1

Modelo cliente/servidor y tipo de páginas

Definiciones

Cliente y servidor web

Cliente: normalmente el navegador (Chrome, Firefox…) que inicia la comunicación. Envía peticiones HTTP al servidor pidiendo recursos.

Servidor web: software (Apache, IIS…) que recibe las peticiones, ejecuta lógica en servidor si hace falta (PHP, JSP, ASP) y devuelve una respuesta (casi siempre HTML).

El navegador nunca ve el código PHP: sólo recibe el HTML generado.

Respuesta compacta de examen: «En la Web el cliente (navegador) envía peticiones HTTP al servidor, que procesa la lógica y devuelve páginas HTML como respuesta».
Test

Página estática

  • Archivo .html guardado tal cual en el servidor.
  • El servidor sólo lee el fichero y lo envía sin modificarlo.
  • El contenido no cambia según el usuario ni según la petición.
Clave

Página dinámica

  • El contenido se genera al vuelo cuando llega la petición.
  • El servidor ejecuta código (PHP, JSP, ASP…), puede consultar una BD.
  • El resultado final es HTML que se envía al navegador.
Frase que tienes que soltar: «En una página dinámica el servidor ejecuta código (PHP, JSP, ASP…), genera HTML y al cliente sólo le llega ese HTML, nunca el código».
Tema 1

Componentes y capas en una web dinámica

Componentes

Elementos básicos

  • Lenguaje de servidor: PHP, JSP, ASP…
  • Servidor web: Apache, IIS…
  • Módulo del lenguaje: por ejemplo, módulo PHP dentro de Apache.
  • Base de datos: MySQL/MariaDB, etc.

El servidor web recibe la petición, detecta que es un .php, delega en PHP, y el resultado vuelve al servidor para ser enviado al cliente.

3 capas

Arquitectura en 3 capas

Presentación

HTML, CSS y JavaScript en el navegador. Interfaz que ve el usuario.

Lógica de negocio

Código PHP en servidor: valida datos, aplica reglas, decide qué mostrar.

Datos

Sistema gestor de BD (MySQL). Guarda usuarios, productos, pedidos, etc.

Si preguntan ventajas: separación de responsabilidades, más fácil de mantener, escalar y reutilizar. Inconveniente: más compleja y requiere más infraestructura.
Tema 1

Lenguajes en servidor e integración con Apache

Repaso

CGI y tecnologías de scripting

  • CGI (Common Gateway Interface): primer mecanismo para ejecutar programas en servidor. Creaba un proceso por petición → poco eficiente → casi en desuso.
  • PHP: software libre, muy usado, se integra en HTML, corre en servidor, soporta programación estructurada y orientada a objetos.
  • ASP: tecnología de Microsoft para páginas dinámicas.
  • JSP/Servlets: Java en servidor. JSP se convierte en servlet y se ejecuta en un contenedor (Tomcat).
Servlets

Contenedor de servlets

Software (como Tomcat) que:

  • Carga los servlets y gestiona su ciclo de vida (init, service, destroy).
  • Recibe peticiones HTTP y las redirige al servlet adecuado.
  • Se encarga de la gestión de recursos y de la concurrencia.
Flujo

Cómo se integra PHP con Apache

  1. El cliente pide http://localhost/proyecto/hola.php.
  2. Apache recibe la petición HTTP.
  3. Detecta la extensión .php y pasa el fichero al intérprete PHP.
  4. PHP ejecuta el script, puede consultar la base de datos.
  5. PHP genera un HTML (o JSON, etc.) como salida.
  6. Apache envía ese HTML al navegador.
Si sale una pregunta de desarrollo, describe este flujo de 4–5 líneas y remata con: «El cliente sólo recibe el HTML resultado de ejecutar el script en servidor».
Tema 1

Herramientas de desarrollo, XAMPP, Node y TypeScript

Herramientas

Navegadores y editores

  • Navegadores: para probar las páginas (DevTools, consola, redes, etc.).
  • Editores ligeros: Notepad++, etc., para archivos sueltos.
  • IDEs: NetBeans, etc., para proyectos grandes (depurar, autocompletar, integrar con servidor).
XAMPP

AMP = Apache + MySQL + PHP

  • XAMPP: paquete que incluye Apache, MySQL/MariaDB, PHP, phpMyAdmin, etc.
  • Desde el panel controlas Apache y MySQL.
  • Carpeta de trabajo típica en Windows: C:\xampp\htdocs.

Para acceder a ejemplo.php dentro de htdocs/proyecto/: http://localhost/proyecto/ejemplo.php.

Extra

Node.js y TypeScript (visión general)

Node.js: entorno de ejecución para JavaScript en servidor (fuera del navegador). Permite crear servidores HTTP y aplicaciones completas sólo con JS.

TypeScript: superconjunto de JavaScript con tipado estático (añade tipos, interfaces, etc.). Se transpila a JavaScript normal, que es lo que realmente ejecutan navegador y Node.

Si cae algo, con decir que «Node permite ejecutar JS en servidor» y «TypeScript añade tipos estáticos a JS y luego se transforma a JS puro» vas sobrado.
Tema 2

Inserción de código PHP, php.ini y variables

PHP / ASP / JSP

Cómo se embebe código de servidor

En PHP, el código se inserta entre etiquetas especiales:

<?php // código PHP ?>

Todo lo que esté dentro se ejecuta en servidor. Fuera es HTML normal.

  • ASP: bloques <% ... %> o
    <script language="VBScript" runat="Server">...</script>.
  • JSP: sintaxis parecida a <% ... %>, aunque en el tema sólo se ve por encima.
Si te preguntan «¿Cómo se inserta código PHP en un HTML?», di que se usa <?php ... ?> y que ese bloque se ejecuta en servidor.
Config

Fichero php.ini y variables básicas

php.ini es el archivo de configuración de PHP. Algunas directivas que salen mucho:

  • file_uploads: activa o no las subidas de ficheros por HTTP.
  • upload_max_filesize: tamaño máximo de fichero subido.
  • max_execution_time: tiempo máximo que puede tardar un script.
  • short_open_tags: permite usar <? ... ?> en lugar de <?php ... ?> (no recomendable).
  • error_reporting: qué errores se muestran.

Variables en PHP:

  • Siempre empiezan por $: $nombre, $edad.
  • Sensibles a mayúsculas/minúsculas ($edad ≠ $Edad).
  • Se crean al asignar un valor: $x = 5;.
Tema 2

Tipos, conversiones, cadenas y operadores en PHP

Tipos

Tipos de datos y conversiones

Tipos básicos en PHP:

  • string → cadenas.
  • integer → enteros.
  • float → decimales.
  • boolean → true / false.
  • array → conjunto de valores.
  • Especiales: object, resource, NULL.

Funciones típicas de comprobación:

  • is_string(), is_int(), is_bool(), is_array(), is_numeric(), is_null().

Conversión de tipos:

$entero = intval($var); $real = floatval($var); $texto = strval($var); $otroInt = (int)$var; // casting
Cadenas

Strings y concatenación

Una cadena es un conjunto de caracteres entre comillas dobles.

$nombre = "Mita"; echo "Hola {$nombre}"; // se recomienda usar llaves $resultado = $texto1 . $texto2; // operador de concatenación

Funciones típicas:

  • str_replace() → sustituir texto.
  • strtolower() / strtoupper() → minúsculas/mayúsculas.
  • strlen() → longitud.
  • ltrim() → quita espacios a la izquierda.
Operadores

Aritméticos, comparación, lógicos

Aritméticos: + - * / % ++ --

Comparación:

  • == (igualdad de valor)
  • === (igualdad de valor y tipo)
  • != o <> (distinto), <, >, <=, >=

Lógicos: &&, ||, !

Asignación: =, +=, -=, *=, /=, .= (concatena y asigna).

Tema 3

Sentencias condicionales y bucles en PHP

Condicionales

if, if-else y switch

if: ejecuta el bloque sólo si se cumple la condición.

if ($edad >= 18) { echo "Mayor de edad"; }

if-else: dos caminos posibles.

if ($edad >= 18) { echo "Mayor"; } else { echo "Menor"; }

switch: cuando una variable puede tomar muchos valores.

switch ($perfil) { case "admin": // ... break; case "user": // ... break; default: // ... }
Bucles

for, while, do-while, foreach

Todos repiten un bloque, pero cambian la forma de control:

  • for: número de repeticiones conocido.
  • while: «mientras que» se cumpla la condición (puede no entrar).
  • do-while: ejecuta el bloque al menos una vez, luego comprueba.
  • foreach: recorre arrays de forma sencilla.
for ($i = 1; $i <= 10; $i++) { echo $i; } while ($i <= 10) { echo $i; $i++; } foreach ($usuarios as $u) { echo $u; }
Recuerda break para salir de un bucle y continue para saltar a la siguiente iteración.
Tema 3

Arrays, matrices y comentarios

Arrays

Crear y trabajar con arrays

Un array guarda varios valores en una sola variable.

// Array asociativo (clave => valor) $notas = array( "Juan" => 7, "Ana" => 9, ); // Array indexado $nums = array(1, 2, 3); $nums[] = 4; // añade al final

Operaciones típicas:

  • unset($array) → borrar todo el array.
  • unset($array[2]) → borrar un elemento.
  • array_merge($a, $b) → unir arrays.
  • array_slice($a, $offset, $length) → extraer parte.
  • sort() / rsort() → ordenar.
Matrices

Arrays multidimensionales

Una matriz es un array que contiene otros arrays.

$matriz = array( "user1" => array("login" => "juan", "perfil" => "admin"), "user2" => array("login" => "ana", "perfil" => "user"), ); echo $matriz["user1"]["login"]; // "juan"
Comentarios

Comentar código PHP

  • // comentario de una línea
  • # también una línea
  • /* comentario multilínea */

En HTML los comentarios son <!-- ... -->, pero no sirven dentro de bloques PHP.

En un caso práctico, comentar bien un bucle o una función suele sumar puntos fáciles.
Tema 4

Funciones en PHP y funciones útiles del lenguaje

Propias

Qué es una función y cómo se define

Una función es un subprograma con un nombre, parámetros opcionales y un bloque de instrucciones. Puede devolver un valor con return.

function sumar($a, $b = 0) { $resultado = $a + $b; return $resultado; } $total = sumar(3, 5); // 8

Paso de parámetros:

  • Por valor: se pasa una copia (comportamiento por defecto).
  • Por referencia: se usa &$param para modificar la variable original.
  • Permiten reutilizar código.
  • Hacen el programa más legible.
  • Facilitan depurar y mantener.
Frase de examen: «Una función en PHP es un subprograma identificado por un nombre, con parámetros opcionales, que agrupa instrucciones y puede devolver un valor mediante return».
Built-in

Funciones ya existentes más usadas

PHP trae muchas funciones. En el tema destacan:

  • Fecha y hora: date(formato[, timestamp]).
  • Aleatorios: rand() o rand($min, $max).
  • Directorios: getcwd() devuelve el directorio actual.
  • Impresión: echo, print, print_r($array).
  • Ficheros: fopen(ruta, modo), fclose(manejador).
  • Búsqueda de ficheros: glob('*.php').
$hoy = date('d/m/Y'); $aleatorio = rand(1, 10); $ficherosPhp = glob('*.php');
Si te piden ejemplos de funciones de PHP, menciona date, rand, getcwd, glob, fopen/fclose y print_r.
Tema 4

Formularios HTML: ACTION, METHOD y name

Estructura

Formulario básico

Un formulario HTML envía datos al servidor cuando el usuario pulsa el botón submit.

<form action='calcula.php' method='get'> Valor 1: <input type='text' name='valor1'> Valor 2: <input type='text' name='valor2'> <input type='submit' value='Aceptar'> </form>
  • action: fichero PHP que procesará los datos.
  • method: get o post.
  • name en cada input: clave para recuperar el valor en PHP.
Clave test

Qué preguntan de los formularios

  • Qué indica action en un formulario.
  • Diferencias entre method='get' y method='post'.
  • Para qué sirve el atributo name en los campos.
  • Qué archivo procesa la información (el del action).
Respuesta rápida: «ACTION indica el script del servidor que recibe los datos, METHOD indica cómo se envían (GET o POST) y NAME identifica cada dato que luego recupero con $_GET o $_POST».
Tema 4

Métodos GET y POST en PHP

GET

Características de GET

  • Los datos viajan en la URL, después de un signo ?.
  • Se ven en la barra del navegador.
  • Hay limitación de longitud.
  • Adecuado para búsquedas y filtros.
// URL que llega al servidor: http://dominio.com/calcula.php?valor1=3&valor2=5 // En calcula.php: $valor1 = $_GET['valor1']; $valor2 = $_GET['valor2']; $resultado = $valor1 + $valor2;
POST

Características de POST

  • Los datos viajan en el cuerpo del mensaje HTTP.
  • No se ven en la URL.
  • Permite enviar más información (formularios largos, contraseñas, etc.).
$usuario = $_POST['login']; $password = $_POST['password']; function validar($u, $p) { if ($u == 'user1') { if ($p == '1234') { echo 'Correcto, puedes pasar!'; } else { echo 'Contraseña incorrecta'; } } else { echo 'Usuario incorrecto'; } }
Diferencia que tienes que soltar: «GET envía los datos en la URL y POST en el cuerpo de la petición. GET es visible y limitado, POST es más adecuado para datos sensibles y formularios grandes».

FAQ

Preguntas típicas de desarrollo (para memorizar)

Explica el recorrido completo de un formulario enviado por POST en PHP.

El usuario rellena el formulario HTML y pulsa el botón de enviar. El navegador envía una petición HTTP POST al servidor con los datos en el cuerpo del mensaje.

El servidor web recibe la petición y la redirige al script indicado en el atributo action. En ese script PHP, los datos se recuperan con la superglobal $_POST usando las claves name de los campos.

El script aplica la lógica necesaria (validaciones, consultas a la base de datos, etc.) y genera una página HTML de respuesta, que se devuelve al navegador.

Diferencias entre GET y POST.
  • GET envía los datos en la URL; POST los envía en el cuerpo de la petición.
  • En GET los datos se ven en la barra del navegador; en POST no.
  • GET está más limitado en tamaño; POST permite formularios grandes.
  • GET se usa para búsquedas y filtros; POST para datos sensibles y modificaciones.
  • En PHP se recuperan con $_GET o $_POST según el método utilizado.
Ventajas de usar funciones propias en un programa PHP.
  • Evitan repetir código: se escribe una vez y se reutiliza.
  • Mejoran la legibilidad: el programa se divide en bloques lógicos con nombre.
  • Facilitan la depuración: los errores se localizan en una función concreta.
  • Permiten reutilizar la misma lógica en varios scripts o proyectos.
Qué es php.ini y menciona algunas directivas importantes.

php.ini es el fichero principal de configuración de PHP. Desde él se controlan aspectos como el manejo de errores, la subida de ficheros o el tiempo máximo de ejecución.

  • file_uploads: activa o desactiva la subida de ficheros.
  • upload_max_filesize: tamaño máximo de cada fichero subido.
  • max_execution_time: segundos máximos de ejecución de un script.
  • short_open_tags: permite o no usar <? en lugar de <?php.
  • error_reporting: define qué tipos de errores se muestran.
